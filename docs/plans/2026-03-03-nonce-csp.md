# Nonce-based CSP Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Replace static hash-based CSP with per-request nonce-based CSP so Next.js inline scripts execute in production.

**Architecture:** Add nonce generation to the existing `proxy.ts`, extract CSP building into a testable helper, pass the nonce to `layout.tsx` via `x-nonce` header. Remove static CSP from `next.config.ts`.

**Tech Stack:** Next.js 16 (proxy.ts), Vitest (unit tests)

**Step 3: Implement `buildCsp` and add nonce to proxy function**

In `web/proxy.ts`, add the `buildCsp` function and modify the `proxy` function to generate a nonce, set `x-nonce` request header, and set `Content-Security-Policy` response header on every `NextResponse.next()` and `NextResponse.redirect()` call.

```ts
export function buildCsp(nonce: string, isDev: boolean): string {
  return [
    "default-src 'self'",
    `script-src 'self' 'nonce-${nonce}' 'strict-dynamic'${isDev ? " 'unsafe-eval'" : ""}`,
    "style-src 'self' 'unsafe-inline'",
    "img-src 'self' blob: data:",
    "font-src 'self'",
    "connect-src 'self'",
    "frame-ancestors 'none'",
    "base-uri 'self'",
    "form-action 'self'",
  ].join("; ");
}
```

In the `proxy` function:
1. Generate nonce at the top: `const nonce = Buffer.from(crypto.randomUUID()).toString("base64")`
2. Build CSP: `const csp = buildCsp(nonce, process.env.NODE_ENV === "development")`
3. For every `NextResponse.next()` call, pass request headers with `x-nonce` set, and set CSP on the response
4. For `NextResponse.redirect()` calls, set CSP on the response

Helper pattern to avoid repetition:

```ts
function withCsp(response: NextResponse, csp: string): NextResponse {
  response.headers.set("Content-Security-Policy", csp);
  return response;
}

function nextWithNonce(request: NextRequest, nonce: string, csp: string): NextResponse {
  const requestHeaders = new Headers(request.headers);
  requestHeaders.set("x-nonce", nonce);
  const response = NextResponse.next({ request: { headers: requestHeaders } });
  return withCsp(response, csp);
}
```

### Task 2: Pass nonce to theme script in layout.tsx

**Files:**
- Modify: `web/app/layout.tsx`

**Step 1: Update layout.tsx**

Read the nonce from the `x-nonce` request header using `headers()` from `next/headers` and pass it to the theme `<script>` tag:

```tsx
import { headers } from "next/headers";

// In RootLayout:
const headersList = await headers();
const nonce = headersList.get("x-nonce") ?? undefined;

// Replace:
<script dangerouslySetInnerHTML={{ __html: themeScript }} />
// With:
<script nonce={nonce} dangerouslySetInnerHTML={{ __html: themeScript }} />
```
